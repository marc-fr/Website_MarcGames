Behind the scene

TRE (Tiny-Render Engine) is a toolkit built on the top of SDL2 and OpenGL. This is the basement of my games. This is not a game-engine, but it helps with mesh operations, UI, shaders (tiny material library), contact algorithms and sound thread.

TRE is open-source and hosted on GitHub: TinyRenderEngine

Some though about C++ feature-sets for games

TRE is using the STL, that uses exceptions for error handling. This has no noticeable impact on the performances, since c++ has a nice paradigm that says: "don't pay for what you don't use". As far as exceptions are considered as nonrecoverable fatal errors, it is not relevant (in a solo-developer point of view) to not use the STL.

Rendering API: modern OpenGL vs. Vulkan

Even though OpenGL is a pretty old rendering API, it is still used in legacy applications and small projects. Using Vulkan gives a great opportunity to work on the CPU code that submits commands to the GPU. For example, Vulkan allows commands to be generated from multiple threads, while OpenGL does not. While my games won't be limited by the rendering CPU thread, OpenGL will still be the default rendering API.

Oriented-Object vs. Data-Driven paradigm for games

The oriented-object paradigm is often adopted by software developers. For generic game engines, this paradigm implies to have an "entity" oriented architecture. It is demonstrated for example in the following talk: Data Driven Entity Component System in C++17. But, at a coarse level, a game is basically a software that transforms data into rendering of frames (video and sound). The data can be split by the usage:

  • User data (mouse, keyboard, joystick, ...)
  • Game data (game state, game logic, physic, ...)
  • Render data (geometry, textures, lighting, ...)
The data-driven paradigm allows better scopes for data, and facilitates parallelism (and noticeable with the rendering thread(s)). For more details, find here some references about data-oriented design in C++:
  • CppCon 2014: Mike Acton "Data-Oriented Design and C++"
  • CppCon 2018: Stoyan Nikolov “OOP Is Dead, Long Live Data-oriented Design”
